# Code Block Syntax Highlighting Design

## Overview

Add syntax highlighting to `CodeBlock.vue` using Shiki, integrated out of the box with no user configuration required.

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Highlighter library | Shiki | VS Code quality themes, fine-grained imports to control bundle size |
| Import strategy | `shiki/core` + JS regex engine | Avoids WASM dependency, tree-shakeable, ~50-100KB gzip for engine + 13 languages |
| Theme | `github-dark` | Matches existing dark code block background style |
| Configuration | None exposed | Single default theme, no props — keeps API simple |
| Loading strategy | Lazy singleton | First highlight triggers initialization, cached thereafter |
| SSR | Client-only init | Highlighter only created on client; SSR falls back to plain text |

## Scope

### Changes

1. **`package.json`** — add `shiki` to `dependencies`
2. **New file: `src/utils/highlighter.ts`** — lazy-initialized Shiki singleton
3. **`CodeBlock.vue`** — replace plain text with Shiki-highlighted HTML
4. **`style.css`** — remove hardcoded code block colors, let Shiki theme own them

### No changes

- `MdRenderer.vue`, `TokenNodes/index.ts`, other token components
- No new props exposed on `StreamBubble`
- No changes to public API

## Detailed Design

### highlighter.ts

- Exports a single `highlightCode(code: string, lang: string): Promise<string>` function
- Uses `shiki/core` entry point with `createJavaScriptRegexEngine` from `shiki/engine/javascript` (no WASM)
- Imports the theme from `@shikijs/themes/github-dark`
- Imports each language grammar individually from `@shikijs/langs-*` packages for tree-shaking
- Supported languages (exported as `SUPPORTED_LANGUAGES` array for easy extension):
  - `javascript`, `typescript`, `python`, `css`, `html`, `json`, `bash`, `java`, `cpp`, `go`, `rust`, `sql`, `markdown`
- First call initializes the highlighter (async), subsequent calls reuse the cached instance
- Client-only: wrapped in `typeof window !== 'undefined'` check; on SSR, returns plain text
- If `lang` is not in the loaded set, falls back to plain text
- Error handling: try/catch wraps the highlight call; on any failure, returns plain `<pre><code>` (current behavior preserved)

### CodeBlock.vue

- Reactive state: `highlightedHtml: Ref<string | null>` (null = not yet highlighted, show plain text)
- Constants: `HIGHLIGHT_DEBOUNCE_MS = 150`
- Watches `token.value` and `token.lang` with debounce:
  - On change, `highlightedHtml` is set to null (instant plain text display)
  - After 150ms of no changes, calls `highlightCode()` and sets `highlightedHtml`
- Template: `v-html` with Shiki output when `highlightedHtml` is not null, plain `{{ token.value }}` otherwise
- Shiki outputs a `<pre><code>` with inline styles, so the existing `.md-code-block` wrapper stays for layout
- Security: `v-html` only receives output from `highlightCode()`, which is generated by Shiki from parsed AST tokens — never raw user input
- Lifecycle: watcher is cleaned up on unmount (`onUnmounted`); if a stale highlight resolves after unmount, the result is discarded

### style.css

- Remove `background` and `color` from `.md-code-block` — Shiki's theme handles these via inline styles
- Keep: `padding`, `border-radius`, `overflow-x`, `margin`, `font-family`, `font-size`, `line-height`
- Remove `.md-code-block code` override (Shiki handles `code` styling internally)

## Streaming Behavior

During streaming, `token.value` grows character by character. The debounce strategy is self-contained within `CodeBlock.vue` — it does not depend on a `streaming` prop from the parent:

1. Every `token.value` change resets the debounce timer and sets `highlightedHtml = null` (plain text)
2. After 150ms of no changes, highlighting is triggered
3. Streaming characters arrive in rapid succession (<50ms apart), so the debounce naturally waits for the burst to end
4. Result: plain text during streaming, highlighted code once the block stabilizes

## Language Fallback

If `token.lang` is missing or not in `SUPPORTED_LANGUAGES`, `highlightCode` returns the code wrapped in `<pre><code>` with no syntax spans — identical to current plain-text rendering.

## Future Considerations

- A light theme option may be needed if code blocks appear in user bubbles (blue background). Not in scope for v1.
- The `SUPPORTED_LANGUAGES` array in `highlighter.ts` makes it straightforward to add more languages later.
